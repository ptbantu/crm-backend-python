2.1 订单层级结构 (3层)
旧结构：客户 -> 订单

新结构：客户 -> 项目分组 (文件夹) -> 具体服务订单

视觉表现：进入客户详情后，优先展示业务文件夹（如：📁公司注册组、📁签证组、📁财税组），而非散乱的订单列表。

2.2 订单打标与属性 (创建时录入)
销售在创建订单时，需定义以下维度，以便后续筛选：

业务性质 (Nature)：🔘新办 🔘变更 🔘续签 🔘注销

业务标签 (Tags)：支持多选（如：第1次、2024年度、加急）。

服务归属：自动或手动归入对应的“项目文件夹”。

2.3 执行进度与时效控制
标准模板：每个服务项设置标准SOP（如：标准时长7天）。

阶段划分：

资料收集中

任务执行中

提交审批中

任务完成

预警机制：系统自动计算预期完成时间，若当前日期超过预期且未完成，标记为“超期预警”。
